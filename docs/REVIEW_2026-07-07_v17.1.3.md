# Deep Review — v17.1.3 ship (work of 2026-07-07)

Generated 2026-07-08 by a 132-agent multi-agent review (map -> adversarial double-check -> independent verify, plus competition/scraping research -> vet). Each finding got a skeptic pass; each opportunity was checked against the actual codebase + the zero-cost rule.

Pool: 35 raw findings -> **32 confirmed/plausible fixes**. 53 raw opportunities -> **47 vetted**. Final ranked list = 14 fixes + 6 improvements.

Coverage gap: the `v3-rescale-flip` domain mapper (the irreversible history-mutation migration) errored on the structured-output retry cap, so that one domain did not get its own adversarial pass. Related one-scale bugs still surfaced via the algo-v3 domain (#3, #5); the migration's idempotency / double-apply-on-restart was not specifically audited. Re-run recommended.

## Final ranked suggestions (1-20)

### Fixes

**1. Redeem-code path decouples an active Stripe subscriber (revenue leak)** — S / high — payments
redeemCode unconditionally sets `subscription_tier='code'` for the redeeming user, including an active recurring Stripe subscriber; the `customer.subscription.deleted` handler only downgrades `WHERE subscription_tier='paid'`, so once tier is 'code' a later cancellation never revokes access. Fix: when a live Stripe sub exists, grant the code as an additive expiry extension and leave tier='paid' (or make the deletion handler also cover 'code' rows from the cancelled sub).
Evidence: `src/auth.js:205` (unconditional UPDATE ... 'code') + `src/auth.js:343` (downgrade guarded by `AND subscription_tier='paid'`).

**2. Conviction-curve timeline leaks the paid capper name + full v3 decomposition to free users** — S / high — pick-privacy
getPickTimeline emits `Resume · ${bd.advocate}` (capper name) plus every v3 component (Consensus, Market, Side lean, Fade, Offset with point values). In `/api/game/:id` the timeline is only nulled for locked ranks 2..paid_rank_max, so the free #1 pick and the public tail ship the capper name (paid-only field) and the proprietary breakdown. Fix: strip advocate names + per-component values for non-paid viewers on every pick.
Evidence: `src/pick_timeline.js:55` (+ steps 56-61) + `index.js:1352-1357`.

**3. Serve the v3 total on /api/pick-history so the public archive is one scale** — S / high — algo-v3-engine
Post-flip, upsertPickHistory writes `pick_history.score` = raw v2 (~50-55) for gold picks, while migrated rows were rescaled to 100-135; the endpoint serves only that raw column, so equal-caliber picks render on two scales. Fix: serve `COALESCE(v3_total, score)` for v3-stamped rows.
Evidence: `index.js:667-670` vs `src/storage.js:709`.

**4. Exclude in-game (live) source picks from capper resume aggregation** — M / high — wave1-scrapers
source_ingest flags in-game picks `live=true` only inside sources_json, never as a column; recomputeCapperRatings has no live filter, so mid-game entries are graded into resume_points, inflating advocate points. Fix: persist the live flag as a capper_history column and add `WHERE is_live=0` to the ratings query.
Evidence: `src/source_ingest.js:88,109,116` + `src/capper_ratings.js:93-97`.

**5. Route pre-game MVP picks through the leak-aware display score** — M / high — algo-v3-engine
`/api/mvp` returns pending pre-game rows with `SELECT m.*` (mvp_picks.score = true v3 total) while the board shows the leak-ramped display score, so a paid user reads the concealed true conviction off the MVP tab. Fix: route pending unresolved MVP rows through the same display-score path, or withhold the number until game start.
Evidence: `index.js:614-621` + `src/mvp.js:8-21`.

**6. Top Games tile selects headline pick by raw v2 score, then shows v3 score** — S / med — algo-v3-engine
`/api/games/top` picks the corner pick with `ORDER BY p.score DESC` (raw v2), then overwrites with `tp.score = v3DisplayScore(tp)`. Under v3 the v2-highest is not the v3-highest, so the strip can feature one pick but label it with another's caliber. Fix: rank the game's picks by v3DisplayScore, then take the max (as globalTopRows already does).
Evidence: `index.js:857-863` + `:882` vs `:846-853`.

**7. Stored XSS in the admin capper leaderboard (name in onclick)** — S / med — admin-security
capperLink builds `arg = escHtml(nm).replace(/'/g,'&#39;')` and injects into `onclick="...showCapperDetail('${arg}')"`. escHtml does not escape single quotes, and inside a double-quoted attribute the browser decodes `&#39;` back to `'` before the JS runs, so a scraped handle with a quote executes JS in the admin session. Fix: JS-escape for attribute context (backslash-escape), or move the name to a data attribute with event delegation.
Evidence: `src/admin.js:295-296` + `:4128-4134` (escHtml omits the single quote).

**8. Disambiguate doubleheader / cross-sport source matching (LIMIT 1 unsafe)** — M / med — wave1-scrapers
findGameByTeams matches team-name fragments with `LIMIT 1`, no sport/date/start-time discriminator, so a game-2 doubleheader pick stores against game 1 and grades on the wrong final; dedup key also lacks game_date. Fix: constrain to the pick's sport, tiebreak on nearest un-started game, return null on ambiguity.
Evidence: `src/source_ingest.js:19-38` + `:99-104`.

**9. Tighten sideOf() so a substring collision can't invert a pick's side** — S / med — wave1-scrapers
Bidirectional substring match tests home first, so a short home fragment inside the away outcome stores the away pick as home and grades inverted. Fix: exact equality on abbr/short, substring only above a min length, null when both match.
Evidence: `src/source_ingest.js:59-61`.

**10. Only credit completed tennis sets on the live scorebug** — S / med — live-engine
live_state marks the in-progress set as won via the `h>a` fallback before ESPN sets a winner flag, so a 3-2 lead in set 2 shows "Sets 2-0". Fix: restrict the fallback to completed sets or require `winner===true`.
Evidence: `src/live_state.js:152-154`.

**11. Gate the spread pick side symmetrically so /live never leaks which side CA holds** — S / med — live-value-pulse
On a no-line soccer spread, the pickless side is skipped but the pick-carrying side still emits a locked stub, so a free user sees `pulses.home_spread` present with away absent and infers CA's side. Fix: emit both spread keys or neither (O/U is already symmetric).
Evidence: `index.js:1590`.

**12. Fix soccer away-spread pulse mixing 3-way and 2-way probabilities at kickoff** — S / low — live-value-pulse
Away-spread pre-game anchor includes the draw leg (`1 - 0.74*share`) while `now` excludes it (soccerTrio.away), showing spurious positive buy-low at kickoff. Fix: derive pre from the same 3-way soccerPreTrio.
Evidence: `index.js:1617`.

**13. Draw the conviction curve on the wiped-game (404) detail-page fallback** — S / low — live-ui
on404 swaps in classicBodyHtml but never calls the drawPickTimeline/drawLockedTeaser block, leaving the canvas blank on shared post-game links. Fix: mirror the rAF draw block from the success path.
Evidence: `public/game-detail.js:1118` (vs ~1088).

**14. Cap re-fetching of permanently-unresolvable pending source rows** — S / low — wave1-scrapers
Pending source picks that never grade are re-queried from ESPN every resolveResults Pass 5 forever. Fix: bound with an attempts counter or age cutoff.
Evidence: `src/results.js` Pass 5.

### Improvements

**15. Ship a privacy-safe "fair price vs book price" chip** — S / high — methodology
scoring_v3 already de-vigs the ML market (devig2) and computes per-pick edge, but pick_privacy strips both. Surface no-vig implied prob, book implied prob, and the gap in cents as a standalone publishable number. Zero new data.
Evidence: `src/scoring_v3.js:37-51`, `:147-150` + `src/pick_privacy.js:19-22`.

**16. Expose tier + confidence badges (Proven / Rated / Building, with sample size) publicly** — S / high — trust
capper_ratings already stamps tier (n>=10/25/50), fade flag, Bayesian-shrunk blends, admin-only today. Show a compact public badge with graded-sample count, keeping assume-chance framing ("Building, 18 graded").
Evidence: `src/capper_ratings.js:213-217`.

**17. Ship free public betting calculators under /tools as an SEO funnel** — S / high — growth
Client-side calculator pages (no-vig, EV, parlay, hold/vig, hedge, Kelly) reusing the de-vig math in scoring_v3.js. SEO plumbing already in place.
Evidence: `src/scoring_v3.js:32-51` + analytics/SEO work.

**18. Add a plain-English "why it's ranked" rationale to free + paid picks** — M / high — ux
Compose a short non-numeric rationale from already-public signals (consensus count, line steam, public-betting contrarian, market edge) without exposing the numeric score or leak state.
Evidence: `src/scoring_v3.js:119-165` + `public/modules/picks.js`.

**19. Publish a public capper leaderboard + SEO-indexable per-capper profile pages** — L / high — growth
Full leaderboard, records, tiers, per-sport strengths, equity curves exist but render only in /admin. Expose a public leaderboard + `/capper/:slug` (reuse detail_page.js), today's pick still paywalled.
Evidence: `src/capper_ratings.js` + capper_history; `src/detail_page.js` pattern; no public route in index.js.

**20. Grade cappers and MVP picks on CLV (beat-the-close), not just W/L and ROI** — M / med — methodology
captureLineAtThreshold + results.js already grade votes vs closing line; capper_ratings computes only units/ROI/resume/tier. Add closing-line-value per capper (avg cents vs close, % beating close).
Evidence: `src/storage.js` captureLineAtThreshold + `src/results.js` vs `src/capper_ratings.js`.
